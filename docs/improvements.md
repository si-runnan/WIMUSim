# PhyNeSim 架构设计说明

**PhyNeSim**（Physics-Neural IMU Simulator）是在原始 WIMUSim 基础上提出的新方法。
本文档总结了相对于原始 WIMUSim（master 分支）所做的三项核心改进：
引入 SMPL 体型模型、加入神经残差修正器，以及从逐帧预测升级为窗口级时序建模。

---

## 一、使用 SMPL 替换 H3.6M

### 原架构的局限

原始 WIMUSim（master 分支）使用 H3.6M 17 关节骨架，姿态来源为 MotionBERT
（从视频估计 3D 关节点坐标）。这种方案存在以下问题：

- **无体型建模**：所有人共用同一副固定骨架，无法反映个体身材差异对 IMU 信号的影响
- **格式转换损耗**：3D 关节点坐标需要额外的逆运动学（IK）求解才能得到关节角，引入额外误差
- **数据生态封闭**：H3.6M 是受限数据集，可用序列有限
- **室内局限**：MotionBERT 在户外视频上泛化性差

### SMPL 的改进

SMPL（Skinned Multi-Person Linear Model）是参数化人体模型，由两组参数描述：

- **β（形状参数，10 维）**：控制体型，高矮胖瘦等个体差异
- **θ（姿态参数，24 × 3 × 3 旋转矩阵）**：控制每帧的关节旋转

对应地，WIMUSim 的 B/D 参数可直接从 SMPL 参数获得，无需额外转换：

```
β  →  compute_B_from_beta()  →  B（骨骼向量，每个人不同）
θ  →  smpl_pose_to_D_orientation()  →  D（关节旋转序列）
```

### 具体好处

| 维度 | H3.6M / MotionBERT | SMPL / HMR2.0 |
|------|-------------------|---------------|
| 体型建模 | 无，所有人同一骨架 | β 参数显式建模个体体型，B 直接从 β 计算 |
| 关节表示 | 3D 坐标，需 IK 转换 | 旋转矩阵，直接输入 WIMUSim，无损转换 |
| 姿态精度 | 关节点位置误差 ~50 mm | 旋转矩阵 + 形状参数，可还原全身几何 |
| 数据生态 | H3.6M 封闭数据集 | AMASS 包含数千小时 SMPL 序列（MoVi、TotalCapture、EMDB 均可用） |
| 视频泛化 | 实验室场景为主 | HMR2.0 在户外视频上也能运行 |
| CPM 意义 | B 混合无物理依据 | β → B 链路清晰，B 混合真实反映跨体型差异 |

**核心收益**：体型参数 β → B 的链路打通，使 CPM 中的 B 混合具有真实的物理含义。
不同身材的人佩戴同位置传感器产生不同信号这一现象，可以在仿真中准确再现。

## 二、加入神经残差修正器

### 纯物理仿真的固有局限

WIMUSim 基于刚体动力学建模，物理模型无法捕捉以下真实效应：

```
真实 IMU 信号
    = 骨骼刚体运动          ← WIMUSim 能建模
    + 软组织晃动            ← 无法建模（皮肤、脂肪随骨骼运动有延迟和形变）
    + 传感器与骨骼的相对位移 ← P 参数假设传感器刚性固定
    + 传感器非线性响应       ← H 参数只建模了线性噪声和偏置
    + 交叉轴灵敏度           ← 硬件参数未涵盖
```

这些误差导致仿真信号与真实 IMU 存在系统性偏差（sim-to-real gap）。

### 神经残差修正器的方案

在 WIMUSim 物理仿真的基础上，训练一个 Transformer 网络来预测物理仿真与
真实信号之间的残差：

```
corrected_imu = phys_imu + NeuralSimulator(pose_6d, phys_imu)
                           ↑
                     只学习残差部分
```

训练数据来自 MoVi 数据集，该数据集同时提供 SMPL 姿态参数和真实 IMU 信号，
是目前少有的大规模配对数据集。

### 残差设计的关键优势

相比让网络直接预测 IMU 信号（端到端方式）：

| 维度 | 端到端预测 | 物理 + 残差修正 |
|------|-----------|----------------|
| 学习难度 | 需从头学习所有物理规律（重力、运动学等） | 物理先验已处理大部分，只学小残差 |
| 数据需求 | 需要大量配对数据才能收敛 | 物理强先验降低数据需求，小数据集也能训练 |
| 可解释性 | 黑盒 | 物理部分完全可解释，网络只负责残差 |
| 泛化性 | 对分布外数据（不同传感器、位置）敏感 | 物理部分提供域无关的先验 |
| 训练稳定性 | 目标信号幅度大，梯度不稳定 | 残差幅度小，训练更稳定 |

训练过程中量化物理基线与神经修正后的对比：

```
Physics-only baseline loss: 0.3841
Epoch  50  train 0.1203  val 0.1389  (+63.8% vs physics)
```

### 时序建模：从逐帧预测到窗口级建模

原始 WIMUSim 基于刚体动力学，每帧 IMU 输出仅由**当前帧**的关节角决定，帧与帧之间完全独立：

```
θ_t  →  WIMUSim  →  imu_t        （只用当前帧，无上下文）
θ_{t+1}  →  WIMUSim  →  imu_{t+1}
```

而真实 IMU 信号中存在大量**跨帧依赖**的时序效应，逐帧预测天然无法捕捉：

| 效应 | 原因 | 逐帧是否可建模 |
|------|------|--------------|
| 软组织晃动 | 传感器相对骨骼的位移有惯性，当前误差取决于前几帧的运动历史 | 否 |
| 传感器动态响应 | 真实硬件有低通滤波特性，输出依赖历史输入 | 否 |
| 快速方向变化 | 单帧无法区分"加速中"和"匀速"，需要上下文 | 否 |
| 信号平滑性 | 相邻帧应连续，孤立帧预测容易出现抖动 | 否 |

PhyNeSim 以**滑动窗口**（默认 128 帧，≈ 1.07 s）为单位进行残差修正。
Transformer 的 self-attention 在窗口内允许任意帧对之间相互作用：

```
[θ_{t-127}, ..., θ_t]  →  Transformer  →  [residual_{t-127}, ..., residual_t]
                              ↑
                     窗口内所有帧相互 attention
```

推理时相邻窗口重叠，重叠区域多次预测取均值，保证边界处的平滑性。

### 作为下游任务的评估意义

神经残差修正器的性能可量化衡量 WIMUSim 的仿真质量：

> 物理仿真越准确，残差越小，网络越容易学习，
> 最终在真实测试集（MoVi subjects 61–90）上的性能越好。

这为 WIMUSim 框架本身提供了一个客观的评估指标。

## 三、三项改进的协同效应

```
MoVi 受试者
    ├── SMPL β  →  compute_B_from_beta()  →  B（真实体型）
    ├── SMPL θ  →  smpl_pose_to_D_orientation()  →  D（真实动作）
    └── 参数识别优化 P、H
              ↓
         CPM 跨受试者混合 B/D/P/H
              ↓
         WIMUSim 物理仿真  →  phys_imu
              ↓
         神经残差修正器（MoVi 配对数据训练）
              ↓
         corrected_imu  ≈  真实 IMU
```

三项改进缺一则效果大打折扣：

- 没有准确的 SMPL → B，物理仿真基线差，残差大，网络难以收敛
- 没有神经修正，物理模型的系统性误差无法消除，下游任务精度受限
- 没有时序窗口，软组织惯性、传感器动态响应等跨帧效应无法建模，残差修正不彻底

SMPL 保证了物理仿真的准确性上限；神经残差修正器弥合 sim-to-real gap；
窗口级时序建模进一步捕捉物理模型无法表达的动态效应。
三者共同作用，使 WIMUSim 生成的虚拟 IMU 数据能够有效用于训练和测试真实场景下的算法。

| 数据集 | 用途 | 原因 |
|--------|------|------|
| MoVi subjects 1–60 | 参数识别 + 神经网络训练 | 同时提供 SMPL fits（F_amass）和真实 Xsens IMU，90 人 × 21 动作，配对数据规模充足 |
| MoVi subjects 61–90 | 测试 | 与训练集同源但受试者不重叠，保证评估的独立性；直接用 `scripts/evaluate.py` 运行 |
